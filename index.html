第一章：了解Web及网络基础

1.1使用HTTP协议访问Web

web页面的呈现：

根据Web浏览器地址栏中指定的URL，Web 浏览器从Web服务器端获取文件资源的信息，从而显示出Web页面

像这种通过发送请求获取的服务器资源的Web浏览器等，都可称为客户端

Web是建立在HTTP协议上通信的

1.2HTTP的诞生

1989年3月,HTTP诞生

GERN的蒂姆·博纳斯·李博士提出一种能让远隔两地的研究者们共享知识的设想，该设想最初的理念是：借助多文档之间相互关联形成的超文本，连成可相互参阅的WWW（World Wide Web ，万维网）

WWW的三项构建技术：

  1. 把SGML（标准通用标记语言）作为页面的文本标记语言的HTML
  2. 作为文档传递协议的HTTP
  3. 指定文档所在地址的URL

1.3 网络基础TCP/IP

计算机与网络设备要相互通信，双方就必须基于相同的方法。

:bookmark_tabs:协议的概念

不同的硬件，操作系统之间的通信，所有的这一切都需要一种规则，这种规则就成为协议（protocol）

协议中存在各式各样的内容。从电缆的规则到IP地址选定方法，寻找异地用户的方法，双方建立通信的顺序，以及Web页面显示需要处理的步骤，等等。

而把这样与互联网相关联的协议集合起来总称为TCP/IP

TCP/IP的两种说法：

  1. TCP是指 TCP和IP两种协议
  2. TCP/IP 是在IP协议的通信过程中，使用到的协议族的统称

一.力扣406

假设有打乱顺序的一群人站成一个队列，数组 people 表示队列中一些人的属性（不一定按顺序）。每个 people[i] = [hi, ki] 表示第 i 个人的身高为 hi ，前面 正好 有 ki 个身高大于或等于 hi 的人。

请你重新构造并返回输入数组 people 所表示的队列。返回的队列应该格式化为数组 queue ，其中 queue[j] = [hj, kj] 是队列中第 j 个人的属性（queue[0] 是排在队列前面的人）。

示例 1：

输入：people = [[7,0],[4,4],[7,1],[5,0],[6,1],[5,2]]
输出：[[5,0],[7,0],[5,2],[6,1],[4,4],[7,1]]
解释：
编号为 0 的人身高为 5 ，没有身高更高或者相同的人排在他前面。
编号为 1 的人身高为 7 ，没有身高更高或者相同的人排在他前面。
编号为 2 的人身高为 5 ，有 2 个身高更高或者相同的人排在他前面，即编号为 0 和 1 的人。
编号为 3 的人身高为 6 ，有 1 个身高更高或者相同的人排在他前面，即编号为 1 的人。
编号为 4 的人身高为 4 ，有 4 个身高更高或者相同的人排在他前面，即编号为 0、1、2、3 的人。
编号为 5 的人身高为 7 ，有 1 个身高更高或者相同的人排在他前面，即编号为 1 的人。
因此 [[5,0],[7,0],[5,2],[6,1],[4,4],[7,1]] 是重新构造后的队列。
示例 2：

输入：people = [[6,0],[5,0],[4,0],[3,2],[2,2],[1,4]]
输出：[[4,0],[5,0],[2,2],[3,2],[1,4],[6,0]]

提示：

1 <= people.length <= 2000
0 <= hi <= 106
0 <= ki < people.length
题目数据确保队列可以被重建

来源：力扣（LeetCode）
链接：https://leetcode-cn.com/problems/queue-reconstruction-by-height

这题的主要思路是：先将数组按照一定规律来排序，然后在进行插入

排序的策略是先按身高h从大到小进行排队，再按k小放前面

然后我们进行插队操作即按k值来进行插入，

我们可以发现当我们根据K值来进行插入的时候，当当前插入节点所需所以位置被占时，只需要将原位置元素向后移一次即可，因为我们已经事先确定了队列是按身高进行排队的，K值相同，身高小的本应该就在前面。

而这就是我们的局部最优，身高高的K先进行插入。

贪心算法

1. 什么是贪心算法

所谓贪心算法就是在解决问题时，不去考虑全局最优解，而是在每一步都取当前的局部最优解。

即贪心算法总是在做目前看来最好的选择，所以贪心算法不能对所有问题都得到最优解。

1. 什么是编程

2. 什么是Java

Java是一门面向对象编程语言，不仅吸取了C++语言的各种优点，还舍弃难以理解的多继承、指针等概念，因此Java语言具有功能强大和简单易用两个特征。

3. jdk1.5后的三大版本

Java SE

Java SE 以前称为 J2SE。它允许开发

和部署在桌面、服务器、嵌入式环境和实时环境中使 用的 Java 应用程序。Java SE 包含了支持 Java

Web 服务开发的类，并为Java EE和Java ME提供基础。

Java EE

Java EE 以前称为 J2EE。企业版本

帮助开发和部署可移植、健壮、可伸缩且安全的服务器 端Java 应用程序。Java EE 是在 Java SE 的

基础上构建的，它提供 Web 服务、组件模型、 管理和通信 API，可以用来实现企业级的面向服务

体系结构（service-oriented architecture，SOA）和 Web2.0应用程序。

Java ME

Java ME 以前称为 J2ME。Java ME 为在移动设备和嵌入式设备（比如手机、PDA、电视 机顶盒和打印机）上运行的应用程序提供一个健壮且灵活的环境。Java ME 包括灵活的用 户界面、健壮的安全模型、许多内置的网络协议以及对可以动态下载的连网和离线应用程序 的丰富支持。基于 Java ME 规范的应用程序只需编写一次，就可以用于许多设备，而且可 以利用每个设备的本机功能。

4.JVM JRE JDK

JVM

JVM指Java Virtual Machine，Java程序运行在虚拟机上，不同的平台有自己的虚拟机，因此Java可以实现跨平台。

JRE

JRE指Java Runtime Environment，包括Java虚拟机和Java程序所需的核心类库等，核心类库主要指是ava.lang包。

JDK

JDK指Java Development Kit，是提供给Java开发人员使用的，其中包括了Java开发工具，也包括JRE。其中开发工具有编译工具javac.exe,打包工具jar.exe等

跨平台性和原理

Java的跨平台性是指Java编写的程序在一次编译后，可以在多个系统平台上运行。

原理：Java程序通过JVM在系统平台上运行，只要该平台上安装了相应的JVM，那么该系统就能运行Java程序。

  5. 支持多线程
  6. 健壮性 ：Java语言的强类型机制 异常处理 垃圾的自动回收等

6. 字节码和采用字节码的好处

字节码： Java源代码经过虚拟机编译后产生的.class文件，不面向任何特定的处理器，只面向虚拟机。

采用字节码的好处：

Java语言通过字节码的方式在一定程度上解决了传统解释型语言执行效率低的问题，同时又保留了解释型语言可移植的特点，所以Java程序运行时比较高效，而且字节码不专对于一种特定的机器，因此Java程序无需重新进行编译便可在多种不同的计算机上运行。

Java程序运行流程

Java源代码 —-》 编译器 ——》.class文件 ——》jvm —》 jvm中的解释器 —–》 机器可执行的二进制机器码 —–》程序运行

7.Java主类 应用程序和小程序的主类有何不同

一个程序可以有多个类但只能有一个主类。

主类是java程序的入口

在Java应用程序中主类指包含main()方法的类，不一定要求是public类

在Java小程序中，这个主类是一个继承子系统类Applet或Applet的子类，必须是public类

8. Java应用程序与小程序之间有哪些差别

应用程序从主线程启动(main()方法)

applet小程序没有main方法，主要是嵌在浏览器页面上运行(调用init()线程或者run()来启动)

9. Java和C++的区别

  1. 都是面向对象的语言都支持封装 继承 多态
  2. Java不提供指针来访问内存，程序内存更加安全
  3. Java类是单继承的，c++支持多重继承，Java中的接口可以实现多继承
  4. Java有自动内存管理机制，不用程序员手动释放无用内存

  3. 建立TCP连接，在发送数据前建立通道 —->三次握手

  5. 响应HTTP请求
  6. 页面渲染
  7. 断开连接 —–》四次挥手

  1. 首先我们需要去找到老师的办公室比如说是班主任室 —–> 输入URL www.班主任室.com

  2. 但是我们只知道班主任室这个名称是找不到的，我们需要把这个名称解析成具体的地址如 三栋三层左拐第一间 —–> 这就是DNS解析(DNS可以一个数据库，里面存放着很多URL和对应的ip地址 )

  3. 我们历经千辛万苦终于到了班主任室的门口，这时我们需要确定老师有空我们能够进去，然后我们敲门说：”报告！”，然后老师正好有空说:”进”，我们得知老师有空然后就说:”那我进来了老师” —–> 这就是建立TCP链接的三次握手

  1. 这时我们终于进来了，然后我们就跟老师说我想请个假 —->发送HTTP请求

  1. 老师了解我们的情况后同意了请假，并且开了假条 —->响应HTTP请求

  1. 然后我们就得到了假条成功出了学校 —> 页面渲染
  2. 然后我们潇洒的去医院了 —–>断开TCP连接 四次挥手
